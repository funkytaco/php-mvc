About php-seed-bootstrap

{{month}} {{day}}, {{year}} by Luis Gonzalez

Components

  • Bootstrap for front-end development
  • Composer for dependency management and project setup
  • whoops for error handling
  • Klein for routing
  • mustache for templating
  • Auryn for dependency injection

If you read through the No Framework tutorial, you can learn how to replace the router (Klein), the template engine (Mustache), the error handler, or the dependency injector (Auryn).

Where does everything go?


php-seed-bootstrap/
├── optional
│   └── themes (3rd party themes I didn't install into the app structure)
├── public/ (your public web folder)
│   ├── assets/ (css, js, et cetera)
│   └── index.php (app entry point)
└── src
    ├── Mocks/
    ├── Renderer/ (templating engine files)
    ├── Traits/ (php 5.4 traits)
    ├── Controllers/
    ├── Static/ (your static error page is in this directory)
    ├── Bootstrap.php (setup your project)
    ├── Dependencies.php (for dependency injection)
    ├── Routes.php (setup your URI endpoints/routes)
    └── Views
        └── Partials (templating include files)
Note: Please make sure to read README.md for further details.
